# DietApp MVP (Dietitian-Client Portal) 🍏

A Flutter-based mobile application MVP designed to digitalize and streamline the communication process between dietitians and their clients. This project focuses on core functionality, fast onboarding, and a custom macro-tracking module.

## 🚀 Key Features

* **Frictionless Onboarding (Guest Login):** Users can access the platform instantly without the barrier of entering an email address or completing lengthy registration forms, ensuring a smooth initial user experience.
* **Direct Messaging Module:** A dedicated communication channel facilitating real-time interaction and progress tracking between dietitians and clients.
* **Custom Macro & Protein Calculator:** Features a built-in macro calculation engine. 
  * *Technical Note:* In this MVP phase, the calculation does not rely on external AI APIs. Instead, it operates on a custom, pre-defined static database (containing specific items like banana, rice, etc.) to calculate protein and meal macros. This allows for controlled testing of the app's internal logic and state management.

## 🛠️ Tech Stack
* **Framework:** Flutter / Dart
* **Architecture:** MVP (Minimum Viable Product) approach focusing on core business logic.

## 🚧 Roadmap & Future Work
While the core communication and basic calculation features are functional, the following improvements are planned:
- [ ] Transitioning the static food database to a dynamic REST API or AI-based image/text recognition system for comprehensive food tracking.
- [ ] Expanding the authentication flow for long-term user data retention.
- [ ] Enhancing the UI/UX for the messaging interface.
